# Lean export (PhysLean)

`OpenMaterials.lean` is generated by `python -m omai.physlean_export` from the
live map. Each node is a PhysLean `Dimension`; each theorem states that an
edge's output dimension equals the product of its input dimensions, proved by
extensionality plus `simp` with PhysLean's `*_mul` lemmas.

**Verified.** The file compiles against PhysLean (Lean 4.29.1): 80 nodes, 11
edge theorems, zero errors. `tests/test_physlean_export.py` re-checks every
identity in Python so a regression fails fast even without a Lean toolchain.

To rebuild the proof:

```
# with elan/lake installed and PhysLean available:
cp lean/OpenMaterials.lean PhysLean/ && cd PhysLean && lake env lean OpenMaterials.lean
```

Scope (Tier 1): the dimensional layer only. Nodes whose dimension uses
amount-of-substance (mole), current, or luminous intensity are omitted, since
PhysLean's `Dimension` has five bases (length, time, mass, charge, temperature)
and no mole base. Extending it upstream is a later tier.
