Git for Science

A substrate for reproducible, compounding scientific knowledge

G. Barbalinardo · UC Davis

Git stored the object, not the printout

Not files. The repository: a structured object with identity and lineage.

That object is why merge, provenance, and collaboration became mechanical.

Science is still pre-git

We ship the paper, the printout of a result.

The object is thrown away.

The creed

  • Physics is the invariant. A code is one representation of it.
  • A result is a structured object with identity and lineage, not a number in a table.
  • A result you cannot re-run is a claim, not yet knowledge.
  • Correctness should be structural, not re-derived by trust.
  • The native substrate for scientific AI is semantic, not token text.
