Git stored the object, not the printout

Not files. The repository: a structured object with identity and lineage.

That object is why merge, provenance, and collaboration became mechanical.

Science is still pre-git

We ship the paper, the printout of a result.

The object is thrown away.

The creed

  • Physics is the invariant. A code is one representation of it.
  • A result is a structured object with identity and lineage, not a number in a table.
  • A result you cannot re-run is a claim, not yet knowledge.
  • Correctness should be structural, not re-derived by trust.
  • The native substrate for scientific AI is semantic, not token text.

The same physics, a dozen codes

kaldo, phono3py, ShengBTE, LAMMPS, VASP, Quantum ESPRESSO.

They never quite agree, and most results are not reproducible.

Meaning is implicit

Knowledge lives in three lossy forms:

prose (rich, not runnable) · code (runs, meaning buried) · data (numbers, no provenance)

None is at once machine-checkable and runnable.

Merge for science, and it already runs

Silicon κ comes out in W/(m·K) automatically, no hand-applied factor.

\[ \kappa^{\alpha\beta} = \frac{1}{V}\sum_{q\nu} c_{q\nu}\, v^{\alpha}_{q\nu}\, v^{\beta}_{q\nu}\, \tau_{q\nu} \]

Total vs. populations + coherences agree to ~10−6 W/(m·K). Molar heat capacity 11.83 J/(K·mol), cross-check with zero residual.
