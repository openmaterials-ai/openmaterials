DAG extension rules — when to add a node vs. a parameter

Edges carry the sympy formula; states are typed places. Different production formulas force different edges, not necessarily different states. When adding a new variant, pick exactly one pattern:

  1. Pattern A — type parameter on the state. Only when the variant changes the gauge type (Observable vs HiddenState) AND the parameterised state is terminal-ish. Example: ThermalConductivity[bte_solver=rta|direct_inverse], [transport_model=lbte|wigner|qhgk].
  2. Pattern B — sibling states, converging edge. Variants with different input chains that combine before a downstream consumer uses them. Example: AnharmonicLinewidth / IsotopicLinewidth / BoundaryLinewidth → sum_linewidths → TotalLinewidth.
  3. Pattern C — shared output, alternative producing edges. Same output type and gauge classification, different production formulas. A small upstream intermediate keeps it acyclic. Example: BareDynamicalMatrix → (apply_nac_correction OR identity_dm) → DynamicalMatrix.

Anti-pattern: type parameter on an intermediate state. Forces every downstream consumer to be parameterised too. See docs/skills/extend_dag.md for the full skill and docs/operator_representation_substrate.tex § "DAG extension rules" for the canonical statement.
