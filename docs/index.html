How the map is built

Three layers, one graph. Structure comes first, evidence attaches to it, and a content-addressed protocol keeps every step honest.

Structure

Codes and theory add the graph

Each physical quantity is a typed node; each symbolic formula is an edge that produces one quantity from others. Codes declare how their outputs map onto the shared layer, units and conventions included.

Evidence

Runs and measurements attach values

Simulation outputs and experimental measurements attach to the same nodes as values. Where several land on one observable, they reconcile there, so agreement and disagreement are both visible and auditable.

Protocol

Content-addressed, log-first, gated

Every node, edge, and value is content-addressed by a hash. Changes append to a log behind validation gates, so the whole store has an identity and a lineage you can check.

Contribute a value

A recorded value is a small JSON file that names a quantity, a material, the number with its units, and where it came from.

1Write the instance file (the shape on the right).
2Add it under docs/data/instances/ and open a pull request.
3The gates validate it against the graph; on merge, the store version advances.

The document appendices cover ingesting a new code and extending the DAG (when to add a node versus a parameter).
