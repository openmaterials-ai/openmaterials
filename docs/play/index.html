A map from code

A sub-map is a list of edges plus the map version they came from. Paste that JSON to render it, or build one on the canvas and Share it: the link carries the edges in its fragment, no server. Node symbols and formulas are pulled live from the map by id.

A paper's lineage

A parsed paper reports quantities; the map knows how each is produced. Load a proposal to see the paper's method dataflow drawn as a graph, with the same lineage lit on the map. Drop a PDF on the learn page to make a proposal of your own.

An experiment as a link

An experiment record is light: the recipe it was asked (a map node when known, else a template with its hyperparameters and setup values) plus optional pointers to heavy artifacts hosted on MaterialsCodeGraph. Paste a record, or drop a .json file (an MCG-served record pastes straight in), to render it, then Copy link: the whole record rides in the #x= fragment, no server, and reopens here. Nothing is stored or uploaded, it is a client-side view. When the recipe names a map node it lights up on the canvas.
