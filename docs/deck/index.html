The same physics, a dozen codes, and they don't agree

Lattice thermal conductivity of one crystal, computed by kaldo, phono3py, ShengBTE, LAMMPS, each with its own inputs, units, gauge, and conventions.

Comparing two of them is expert, manual work. Most published computational results never reproduce.

The physics is buried in the code

Units, broadening, Brillouin-zone sums, array layouts, gauge choices: the meaning lives in scripts and tribal knowledge, not in any shared, checkable form.

So reconciliation and reproducibility stay artisanal.

Make the physics the invariant; the code a representation

A typed graph: nodes are physical quantities (ω, D, Γ, c, κ…), edges are the symbolic formulas relating them. Each code is an adapter that pins its outputs onto the shared layer, units and conventions declared.

\[ \kappa^{\alpha\beta} = \frac{1}{V}\sum_{q\nu} c_{q\nu}\, v^{\alpha}_{q\nu} v^{\beta}_{q\nu}\, \tau_{q\nu} \]

Reconciliation becomes mechanical

  • Gauge-invariant observables must agree across codes; gauge-dependent intermediates need not, a typed distinction.
  • Units reconcile automatically from declared dimensions.
  • Cross-code comparison is a structural operation on the graph, not a feat of translation.

Real cross-code result

Silicon, Tersoff potential, 8×8×8 mesh, 300 K, computed through the framework:
RTA     kaldo 19.46 · phono3py 16.74
direct   kaldo 26.91 · phono3py 24.30   W/m·K

The ~15% gap is real, and the substrate localizes it: the broadening convention on the scattering rates (Gaussian vs tetrahedron). Reproducibility you can audit, not argue about.

An open, forkable database of physics

Variables + formulas + attached values (simulations and measurements), reconciled at the observables. git clone to run locally; pull request to append.

And the same substrate is what scientific AI will need

Agents that act on typed physics instead of text; an invalid step becomes a type error, not a silent wrong number; redundant routes are a built-in verifier. The horizon, and lattice transport is the proving ground.

Where it goes

  • More codes and domains: every adapter added makes the whole graph more cross-validated.
  • Symbolic composition of error: approximation and discretization, propagated across the graph.
  • Ingesting whole papers, not just codes: published results as objects you can re-run and check.
